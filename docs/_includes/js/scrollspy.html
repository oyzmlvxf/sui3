使用方法

依赖 Bootstrap 的导航组件

滚动监听插件依赖 Bootstrap nav component用于高亮显示当前激活的链接。

需要可分解的具有ID的目标

导航条必须有可分解的具有ID的目标，例如<a href="#home">home</a> 必须与DOM中的形如<div id="home"></div>的元素相对应.

不是:visible目标将会被忽略

不是:visible according to jQuery 的目标元素将会被忽略，与它们对应的导航条条目也不会高亮。

需要相对定位

无论何种实现方式，滚动监听都需要被监听的组件是position: relative;属性，大多数时候我们都是在监听<body>.

通过data属性调用

想要轻松的给你的导航条增加滚动监听行为，增加data-spy="scroll" 到你想要监听的元素上(大多数情况会是<body>). 然后增加data-target 属性，值是任意Bootstrapnav组件的父元素的ID或class. {% highlight css %} body { position: relative; } {% endhighlight %} {% highlight html %} ...

    ...
... {% endhighlight %}

通过javascript

在CSS中加入position: relative;之后,通过javascript调用滚动监听

{% highlight js %} $('body').scrollspy({ target: '.navbar-example' }) {% endhighlight %}

方法

.scrollspy('refresh')

当使用滚动监听插件的同时在 DOM 中添加或删除元素后，你需要像下面这样调用此刷新（ refresh） 方法：

{% highlight js %} $('[data-spy="scroll"]').each(function () { var $spy = $(this).scrollspy('refresh') }) {% endhighlight %}

参数

可以通过 data 属性或 JavaScript 传递参数。对于 data 属性，其名称是将参数名附着到data-后面组成,例如data-offset="".

Name type default description
offset number 10 计算滚动位置相对于顶部的偏移量（像素数）

事件

Event Type Description
activate.bs.scrollspy 每当一个新条目被激活后都将由滚动监听插件触发此事件。
{% highlight js %} $('#myScrollspy').on('activate.bs.scrollspy', function () { // do something… }) {% endhighlight %}
