选择性加入的功能

出于性能的考虑，tooltip和popover的数据api都是选择加入的，这意味着 ，你必须受到初始化它们。

其中一种初始化tooltips的方法是选中它们的data-toggle 属性:

{% highlight js %} $(function () { $('[data-toggle="tooltip"]').tooltip() }) {% endhighlight %}

在按钮组和输入组中的button需要特殊的设置才能初始化

当在.btn-group 或者 .input-group, 中使用tooltips时，你需要指明 container: 'body'来避免不可预见的副作用（诸如元素变得异常大，或者当tooltips触发时失去圆角.

不要试图在隐藏的元素中展现tooltips

调用 $(...).tooltip('show') 当目标元素样式中 display: none; 会导致tooltips出现在错误的位置上

在禁用元素的上的tooltips需要包裹元素才能使用

想要把tooltips增加到 disabled或者.disabled 元素上, 需要给其放到一个<div>元素的内部， 然后在<div> 上应用tooltips.

用法

tooltips组件根据需求生成内容和标记，默认情况是把tooltips放到触发它们的元素后面。

使用js触发tooltip

{% highlight js %} $('#example').tooltip(options) {% endhighlight %}

标记

tooltip需要的标记仅仅是一个data 属性和 标题 在你想要增加tooltip的html元素上。尽管tooltips需要一个位置，但它生成的标记也很简单，（默认被设置到组件top).

多行链接

当你想在一个包含多行的超链接上增加一个tooltip时，tooltip组件的默认的行为是垂直居中。增加 white-space: nowrap; 在锚上可以避免这种情况。

可选项

参数可以使用data属性或者js传递。如果是用data属性，需要将参数名怎急啊到data-, 例如data-animation="".

Name Type Default Description
animation boolean true 在tooltip上增加css fade的动画效果
container string | false false

把tooltip增加到一个具体的元素上。 例如: container: 'body'.该参数的使用允许我们把tooltip放置在触发元素的附近，从而防止在窗口大小变化是tooltip的显示变得异常。
delay number | object 0

延长显示和隐藏tooltip的时间(ms)，不能应用到手动触发类型

当使用数字作为参数时，延时效果会同时应用到hide/show

对象结构是: delay: { "show": 500, "hide": 100 }
html boolean false 插入html到tooltip，如果是false的话，jquery的text方法将会被用于插入。使用text能在一定程度上防止xs s攻击.
placement string | function 'top'

如何定位tooltip - top | bottom | left | right | auto.
当指明auto时, 将会动态重新定位tooltip。例如,如果指定"auto left"，如果可能的话，tooltip将会放置到左侧，否则会放置到右侧.

当使用函数决定位置时，需要将tooltip元素作为第一个参数，触发元素作为第二个参数。 this 指向tooltip的实例
selector string false If a selector is provided, tooltip objects will be delegated to the specified targets. In practice, this is used to enable dynamic HTML content to have tooltips added. See this and an informative example.
template string '<div class="tooltip" role="tooltip"><div class="tooltip-arrow"></div><div class="tooltip-inner"></div></div>'

Base HTML to use when creating the tooltip.

The tooltip's title will be injected into the .tooltip-inner.

.tooltip-arrow will become the tooltip's arrow.

The outermost wrapper element should have the .tooltip class.
title string | function ''

Default title value if title attribute isn't present.

If a function is given, it will be called with its this reference set to the element that the tooltip is attached to.
trigger string 'hover focus' How tooltip is triggered - click | hover | focus | manual. You may pass multiple triggers; separate them with a space.
viewport string | object { selector: 'body', padding: 0 }

Keeps the tooltip within the bounds of this element. Example: viewport: '#viewport' or { "selector": "#viewport", "padding": 0 }

Data attributes for individual tooltips

Options for individual tooltips can alternatively be specified through the use of data attributes, as explained above.

Methods

$().tooltip(options)

Attaches a tooltip handler to an element collection.

.tooltip('show')

Reveals an element's tooltip. Tooltips with zero-length titles are never displayed.

{% highlight js %}$('#element').tooltip('show'){% endhighlight %}

.tooltip('hide')

Hides an element's tooltip.

{% highlight js %}$('#element').tooltip('hide'){% endhighlight %}

.tooltip('toggle')

Toggles an element's tooltip.

{% highlight js %}$('#element').tooltip('toggle'){% endhighlight %}

.tooltip('destroy')

Hides and destroys an element's tooltip.

{% highlight js %}$('#element').tooltip('destroy'){% endhighlight %}

Events

Event Type Description
show This event fires immediately when the show instance method is called.
shown This event is fired when the tooltip has been made visible to the user (will wait for CSS transitions to complete).
hide This event is fired immediately when the hide instance method has been called.
hidden This event is fired when the tooltip has finished being hidden from the user (will wait for CSS transitions to complete).
