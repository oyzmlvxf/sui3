用法

通过data属性或者javascript方式，下拉菜单组件会通过切换父列表项中的.open样式来切换隐藏内容。

在移动设备上，打开一个下拉菜单会增加.dropdown-backdrop作为一个用来点击菜单外来关闭下拉菜单的区域。 这意味着在移动端从一个打开菜单切换到另外一个菜单需要一次额外的点击.

注意:关闭下拉菜单依赖于data-toggle="dropdown"，因此尽可能的使用该属性是比较明智的做法.

使用javascript

通过JavaScript调用下拉菜单:

{% highlight js %} $('.dropdown-toggle').dropdown() {% endhighlight %}

依旧需要data-toggle="dropdown"

无论是通过js方式还是data属性的方式调用下拉菜单 data-toggle="dropdown"总是需要出现在下拉菜单的触发元素上

参数

无

方法

$().dropdown('toggle')

Toggles the dropdown menu of a given navbar or tabbed navigation.

事件

所有的下拉菜单事件都会在 .dropdown-menu的父元素上触发。

所有的下拉菜单事件都有一个relatedTarget 属性, 它的值是一个切换中的锚点

Event Type Description
show 在show实例方法被调用后，事件即刻触发。
shown 在下拉菜单对用户可见后，事件会即刻触发(将会等待CSS动画完成).
hide 事件会在调用hide实例方法后即刻触发
hidden 事件会在下拉菜单对用户不可见时触发(将会等待CSS动画完成).
{% highlight js %} $('#myDropdown').on('show', function () { // do something… }) {% endhighlight %}
