导航栏

默认导航栏

导航栏可以是一个响应式的组件，可以用来放在你的网站的顶部。它在移动设备上会折叠，在宽屏设备上会水平显示

导航栏中的链接目前不支持自适应宽度.

内容溢出

因为我们不知道你的导航栏中到底有多少内容，所以有可能会变成两行。你可以这样解决这个问题：

  1. 减少里面的内容
  2. 用 响应式工具类 在不同的宽度下隐藏一部分内容
  3. 改变导航栏在折叠模式和水平模式的切换点。你可以定制@grid-float-breakpoint变量或者增加自己的媒体查询代码

需要JS插件

如果JS被禁用了，在窄屏上是无法自动折叠的。

自动折叠功能需要 折叠插件.

修改折叠点

当viewport宽度小于 @grid-float-breakpoint 时，导航栏会折叠。当大于等于 时，导航栏会展开成水平模式。你可以通过修改less代码来修改这个值。默认的值是 768px （最小的平板电脑的屏幕宽度）

可访问性

请确保导航栏的容器是 <nav> 元素。如果你用了一个更通用的容器，比如 <div>, 那么你需要显式加上 role="navigation" 属性，不然屏幕阅读器会无法理解。

There are some caveats regarding using form controls within fixed elements on mobile devices. See our browser support docs for details.

总是使用label

如果你不为表单中的每一个input增加label的话，将会使屏幕阅读器在分析时遇到困难。对于这些行内表单, 可以使用 .sr-only隐藏掉labels. 还有一些替代方法可以解决屏幕阅读器这类技术遇到的麻烦，比如使用aria-label, aria-labelledby 或者 title 属性. 如果这些都没有提供的话，屏幕阅读器可能会使用placeholder 属性, 前提是如果它存在的话。 但是要注意使用placeholder作为其他标志的替代方法是不明智的.

组件对齐

使用 .navbar-left 或者 .navbar-right 工具类来对齐链接、表单、按钮或者文字。这些类通过CSS 浮动来实现特定方向的对齐。比如你想对齐几个链接，把它们放到 <ul> 中并且加上工具类即可

这两个类是 .pull-left 和 .pull-right 的mixin，但是，他们通过媒体查询限制在了特定的屏幕尺寸内。

右对齐多个元素

导航栏现在如果有多个 .navbar-right 类，可能会导致问题。因为我们在最后一个 .navbar-right 元素上用了负margin。所以如果有多个元素用了这个类，可能会导致问题

我们会在 v4 版本中修复这个问题
