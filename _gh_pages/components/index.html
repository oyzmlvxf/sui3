额外的定位

因为下拉菜单是通过css自动定位的，所以如果你的容器设置了 overflow: hidden;的话，可能会无法显示出来。

不建议使用 .pull-right 来实现右对齐

As of v3.1.0, we've deprecated .pull-right on dropdown menus. To right-align a menu, use .dropdown-menu-right. Right-aligned nav components in the navbar use a mixin version of this class to automatically align the menu. To override it, use .dropdown-menu-left.

按钮组

把多个按钮放在同一行变成按钮组，并且可以通过额外的JS结合单选/复选框来实现选中效果按钮插件.

按钮组中显示 Tooltips & popovers

在按钮组中显示 tooltip 或者 popover 的时候，你需要设置额外的 container: "body"来防止触发的时候产生一些副作用（比如大小变化，或者圆角不见了）

保证正确的 role 并且要提供一个label

为了保证辅助技术 - 比如屏幕阅读器 - 能正确理解按钮组，需要提供一个正确的 role 属性。对按钮组来说，需要提供 role='group'，对工具栏来说，需要提供 role='toolbar'

有一种例外情况就是按钮组里面只有一个按钮(比如 justified button groups with <button> elements) 或者 下拉菜单.

并且，按钮组和工具栏需要有额外的具体的明确的label，否则即使设置了正确的role，很多辅助技术还是不会显示他们。在这里的例子中，我们使用了 aria-label，不过你也可以用 aria-labelledby。

基本用法

把多个 .btn 放入 .btn-group 中。

<div class="btn-group" role="group" aria-label="...">
  <button type="button" class="btn btn-default">Left</button>
  <button type="button" class="btn btn-default">Middle</button>
  <button type="button" class="btn btn-default">Right</button>
</div>

按钮工具栏

<div class="btn-toolbar" role="toolbar" aria-label="...">

输入框组

通过在前边或者后边增加文本或者按钮来拓展表单控件<input> 。在 .input-group 中使用 .input-group-addon 可以把其他元素添加到 .form-control 中.

只有 <input>s 才行

不要用 <select>，因为他们在 WebKit 浏览器中无法完美渲染

不要用 <textarea>，因为有些情况下他们的 rows 会有问题.

输入框组中的 tooltip 和 popover 需要额外的特殊设置

在 .input-group 中使用 tooltip 或者 popover 的时候, 你必须指定 container: 'body' 容器来避免一些副作用（比如tooltip或者popover触发的时候导致元素尺寸变化或者圆角问题）.

不要和其他组件混合使用

不要把 form-group 或者 grid 直接应用在 input-group 上. 而是应该在把 input-group 嵌套在它们里面.

总是用label

Screen readers will have trouble with your forms if you don't include a label for every input. For these input groups, ensure that any additional label or functionality is conveyed to assistive technologies.

The exact technique to be used (<label> elements hidden using the .sr-only class, or use of the aria-label, aria-labelledby, aria-describedby, title or placeholder attribute) and what additional information will need to be conveyed will vary depending on the exact type of interface widget you're implementing. 默认导航栏

导航栏可以是一个响应式的组件，可以用来放在你的网站的顶部。它在移动设备上会折叠，在宽屏设备上会水平显示

导航栏中的链接目前不支持自适应宽度.

内容溢出

因为我们不知道你的导航栏中到底有多少内容，所以有可能会变成两行。你可以这样解决这个问题：

  1. 减少里面的内容
  2. 用 响应式工具类 在不同的宽度下隐藏一部分内容
  3. 改变导航栏在折叠模式和水平模式的切换点。你可以定制@grid-float-breakpoint变量或者增加自己的媒体查询代码

需要JS插件

如果JS被禁用了，在窄屏上是无法自动折叠的。

自动折叠功能需要 折叠插件.

修改折叠点

当viewport宽度小于 @grid-float-breakpoint 时，导航栏会折叠。当大于等于 时，导航栏会展开成水平模式。你可以通过修改less代码来修改这个值。默认的值是 768px （最小的平板电脑的屏幕宽度）

可访问性

响应式嵌入对象

这是一个用来显示视频、幻灯片等的容器，他在不同的设备上都会保证正确的长宽比来缩放。我们默认提供了 16:9 和 4:3 两种比例。

Rules are directly applied to <iframe>, <embed>, <video>, and <object> elements; optionally use an explicit descendant class .embed-responsive-item when you want to match the styling for other attributes.

这个容器的样式默认就对 <iframe>, <embed>, <video>, 和 <object> 元素生效，如果你想对其他元素生效，你可以给他们加上一个 .embed-responsive-item 类。

注意！你不需要给你的 <iframe> 加上 frameborder="0"，因为我们已经帮你做了。

Previous Next 
<!-- 16:9 aspect ratio -->
<div class="embed-responsive embed-responsive-16by9">
  <iframe class="embed-responsive-item" src="..."></iframe>
</div>

<!-- 4:3 aspect ratio -->
<div class="embed-responsive embed-responsive-4by3">
  <iframe class="embed-responsive-item" src="..."></iframe>
